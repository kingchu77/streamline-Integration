Order Upload via Stream API

Overview

Create Stream orders through the secure API (not WebXL) for greater customization, fewer errors, and faster throughput. Weight, cube, and extended attributes are supported to improve auto-planning accuracy.

Features

Order upload to Stream

Orders created in Sage 50 automatically flow into Stream's TMS over secure API calls so planning can start immediately.

Automatic tracking updates

Stream pushes tracking numbers and delivery confirmations back into Sage 50, keeping accounts and customer service in sync with live shipment status.

Error logging

Failed uploads - such as missing data - are logged for instant visibility, so issues can be reviewed and resolved without guesswork.

Process

1

Export sales orders from Sage 50

New sales orders are exported from Sage 50 and queued for transfer to Stream, ready for logistics planning.

2

Upload to Stream via API

Orders are sent to Stream's API using your preferred field mappings over a secure connection, fitting your Stream setup without exposing sensitive data.

3

Update Sage 50 with tracking

Once uploaded, the Sage 50 sales order is updated with the Stream response - tracking link, consignment number, and tracking ID - so status stays in sync.
